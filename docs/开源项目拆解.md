# AI 搜索开源项目拆解

> 基于《AI搜索项目（二）》课程内容整理

---

## 一、课程概览

本节课拆解了三个 AI 搜索开源项目，从简单到复杂递进：

| 项目 | 复杂度 | 特点 |
|------|--------|------|
| **deep research** | ⭐ 简单 | 多轮搜索，基础澄清 |
| **open deep research** | ⭐⭐ 中等 | 多 Agent 协作，主调度模式 |
| **Flow** | ⭐⭐⭐ 复杂 | 交互最好，功能最全 |

---

## 二、项目一：deep research

### 2.1 项目简介

- **出现时间**：2024 年年中 / 2025 年年初（ChatGPT 上线 deep research 后）
- **本质**：Agent + 多轮搜索
- **技术栈**：纯前端，Tavily 搜索 + OpenAI API

### 2.2 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│                    deep research 流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│    用户提问                                                   │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────┐                                            │
│  │ 生成澄清问题 │  (可选，用户可跳过)                         │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                            │
│  │ 生成搜索查询 │  (JSON 格式)                               │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│    ┌────────┐                                               │
│    │ 循环：  │  ◄───────┐                                   │
│    │ 执行搜索 │          │                                   │
│    └───┬────┘          │                                   │
│        │               │                                   │
│        ▼               │                                   │
│  ┌─────────────┐       │                                   │
│  │ 内容处理     │       │                                   │
│  │ (提取学习成果)│       │                                   │
│  └──────┬──────┘       │                                   │
│         │               │                                   │
│         ▼               │                                   │
│  ┌─────────────┐       │                                   │
│  │ 生成后续查询 │ ──────┘ (结果不够)                        │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼ (结果充足)                                         │
│  ┌─────────────┐                                            │
│  │ 生成最终报告 │                                            │
│  └─────────────┘                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 提示词结构

| 提示词 | 作用 |
|--------|------|
| 系统提示词 | 定义角色，告诉模型可以研究时间相关的主题 |
| 澄清问题生成 | 向用户提问，最多返回 N 个问题 |
| 搜索查询生成 | 生成一系列检索关键词 |
| 内容处理/学习 | 从搜索结果中提取学习成果 |
| 报告生成 | 基于所有学习成果生成最终报告 |

### 2.4 关键发现

**优点**：
- 结构清晰，易于理解
- 提示词写得好，给模型留退路（"如果已经很清晰，可以返回更少"）

**缺点**：
- 澄清问题是写死的，即使问题清晰也会问
- 没有把上一轮学习成果传给下一轮，可能导致重复

**可优化点**：
- 澄清环节应该有布尔值判断，不需要时直接跳过
- 学习成果应该持续追加到 context，避免重复

## 三、项目二：open deep research

### 3.1 项目简介

- **特点**：多 Agent 协作，有主调度模型
- **架构**：一个主管 + 多个研究助理

### 3.2 核心流程

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│                                                              │

│       │                                                      │
│       ▼                                                      │
│  ┌─────────────┐                                            │
│  │ 向用户澄清   │                                            │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                            │
│  │ 转换为研究主题 │                                            │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────────────┐                                    │
│  │   主调度 (Lead)      │                                    │
│  │   研究主管            │                                    │
│  └──────────┬──────────┘                                    │
│             │                                                │
│      ┌──────┴──────┐                                         │
│      ▼             ▼                                         │
│  ┌─────────┐  ┌─────────┐                                   │
│  │研究助理1│  │研究助理2│  (并行工作)                        │
│  │搜索资料 │  │搜索资料 │                                   │
│  └────┬────┘  └────┬────┘                                   │
│       │            │                                         │
│       └────┬───────┘                                         │
│            ▼                                                │
│  ┌─────────────┐                                            │
│  │ 压缩结果     │  (Compress Agent)                         │
│  │ 提取关键信息 │                                            │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                            │
│  │ 格式化处理   │                                            │
│  └──────┬──────┘                                            │
│         │                                                     │

│  ┌─────────────┐                                            │
│  │ 读取网页摘要 │  (真正读网页内容，不依赖摘要)               │
│  └──────┬──────┘                                            │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                            │
│  │ 生成最终报告 │                                            │
│  └─────────────┘                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 Agent 角色

| Agent | 职责 |
|-------|------|
| 澄清 Agent | 向用户提问，澄清需求 |
| 主题转换 Agent | 把用户消息转为研究主题 |
| 主调度 (Lead Researcher) | 分配任务给各助理 |
| 研究助理 (Research Assistant) | 使用工具搜索资料 |
| 压缩 Agent | 提取关键信息，压缩结果 |
| 格式化 Agent | 整理信息格式 |
| 网页摘要 Agent | 真正读取网页内容 |

### 3.4 关键发现

**优点**：
- 多 Agent 并行工作，效率高
- 有压缩机制，避免上下文撑爆
- 真正读网页内容，不被营销摘要欺骗

**核心差异**：
- deep research：所有结果塞给模型，容易撑爆上下文
- open deep research：每轮压缩结果，保持上下文可控

### 4.3 前端交互

**侧边栏设计**：
- **Activities 标签**：实时显示 Agent 调研过程中的搜索结果
- **Report 标签**：流式输出研究报告

**技术实现**：
- 使用 Zustand 集中管理状态
- 通过 SSE 流式传输执行过程
- 使用 Framer Motion 添加动画效果

### 4.4 关键发现

**优点**：
- 交互体验最好
- 支持人工审核
- 可扩展输出（播客、PPT）

**缺点**：
- 后端配置复杂
- 部分提示词有 bug（如要求模型必须输出完整信息，容易导致幻觉）

## 五、三个项目对比

### 5.1 复杂度对比

| 维度 | deep research | open deep research | Flow |
|------|---------------|-------------------|------|
| 代码行数 | 少 | 中 | 多 |
| 提示词数量 | 6 个 | 24 个 | 30+ 个 |
| Agent 数量 | 1 个 | 多个 | 多个 |
| 交互复杂度 | 低 | 中 | 高 |

### 5.2 功能对比

| 功能 | deep research | open deep research | Flow |
|------|---------------|-------------------|------|
| 澄清问题 | ✅ 写死 | ✅ | ✅ 智能判断 |
| 多轮搜索 | ✅ | ✅ | ✅ |

## 六、学习要点

### 6.1 提示词设计原则

1. **给模型留退路**：使用"如果已经很清晰，可以返回更少"
2. **持续追加上下文**：避免多轮之间信息重复
3. **不要写死**：用布尔值判断是否需要某个环节

### 6.2 架构设计要点

1. **结果压缩**：避免上下文撑爆
2. **真正读网页**：不被营销摘要欺骗
3. **流式输出**：提升用户体验

### 6.3 产品设计要点

1. **用户爽是第一位**：不要让用户不爽（过度澄清）
2. **交互体验**：侧边栏展示进度
3. **人工介入**：关键时刻让人审核

## 七、实战建议

### 如何拆解开源项目

**方法一：直接让 AI 复现**
```
我给你一个开源代码，找到某个功能的实现逻辑，
在我当前项目里复现它。
```
→ 优点：快速完成
→ 缺点：学不到东西

**方法二：先学习再复用**
```
分析这个项目的前端交互，总结一个标准化提示词，
未来可以复用这个提示词生成类似代码。
```
→ 优点：学到可复用的知识
→ 缺点：花时间

### 7.2 从简单到复杂

1. **先看 deep research**：理解基础流程
2. **再看 open deep research**：理解多 Agent 协作
3. **最后看 Flow**：学习最佳交互实践
