## 一、AI 搜索的本质

### 1.1 核心原理

AI 搜索本质上就是 **RAG（检索增强生成）**，区别在于检索不在内部知识库，而在外部搜索引擎。

**基础流程：**
```
用户提问 → AI 拆解关键词 → 调用搜索引擎 → 获取结果 → 拼接上下文 → AI 生成答案
```

**Agent 能力增强：**
所有 AI 搜索产品都是在基础流程上增加 Agent 能力：
- AI 根据输入生成多组检索词
- 拿到结果后审视质量
- 质量不行就再生成一组关键词，循环搜索
- 最后汇总所有结果生成分析报告

### 1.2 两大核心价值

| 价值 | 目标用户 | 解决的问题 |
|------|----------|------------|
| **降低信息加工成本** | 咨询顾问、品牌从业者、内容创作者 | 节省人工阅读文章、汇总信息的时间 |
| **帮助不会搜索的人** | 普通用户（更大群体） | 不会抽象关键词、不懂搜索语法（AND/OR/-排除） |

**关键洞察**：AI 搜索本质上是「搜索最佳实践的产品化」。

## 四、行业痛点

### 五大痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **脏数据问题** | 不筛选结果，把所有检索结果塞给模型 | 影响最终内容质量 |
| **流程太快** | 无法像真实研究那样深入探索 | 只能做到综述 |
| **前端体验差** | 复杂交互让用户头大 | 用户体验糟糕 |
| **无法离线** | 关闭页面任务中断 | 使用受限 |
| **结果质量不稳定** | 依赖搜索 API 返回的摘要质量 | 可能生成错误答案 |

### 根本原因

**大部分 AI 搜索产品实现逻辑非常简单：**
1. 拆关键词
2. 调用搜索 API
3. 把结果塞给模型
4. 生成报告

**真正复杂的地方：**
- 结果筛选（Wisper 是少数做的）
- 深入阅读（Kimi 会用浏览器真正读网页）
- 多轮验证（避免脏数据进入上下文）

## 五、用户分层

| 用户类型 | 代表产品 | 需求特点 |
|----------|----------|----------|
| **浅层用户** | 秘塔、Deepseek | 快速获取信息概览，了解知识面 |
| **深度用户** | Kimi、Wisper | 高质量结果，愿意花时间深入研究 |
| **学术用户** | Consensus、Aminer | 论文检索，学术资源 |

**重要认知**：真正做研究时，用户会针对某个关键词花一上午深入研究——边搜边学边发现新关键词，这个过程太快了反而没用。

## 六、产品机会

### 市场空白

1. **平衡型产品**：介于极简和深度之间，兼顾速度和质量
2. **兜底机制**：大部分产品没有搜索失败的兜底方案
3. **开源可部署**：大部分产品闭源，用户无法自部署
4. **学术 + 通用**：单一场景产品较多，全场景产品少

### 设计原则

基于竞品分析，一个好的 AI 搜索产品应该：

1. **有兜底机制**：搜索失败时仍能返回有用信息
2. **结果筛选**：验证搜索结果质量，过滤脏数据
3. **流式输出**：实时展示搜索进度，让用户看到思考过程
4. **适度深入**：不要太快，也不要太慢，平衡速度和质量
5. **简洁 UI**：避免复杂交互，降低使用门槛
6. **来源可追溯**：所有答案都标注来源
