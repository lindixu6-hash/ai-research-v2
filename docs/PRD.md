## 1. 产品概述

### 1.1 产品定位

**AI 搜索引擎**是一款深度研究型智能搜索工具，专注于为用户提供**精准、透明、有据可查**的答案。

与 ChatGPT 等通用 AI 不同，我们：
- ✅ 实时联网搜索，信息不过时
- ✅ 每个答案标注来源，可验证
- ✅ 根据问题类型自适应调整回答深度

### 1.2 目标用户

| 用户类型 | 痛点 | 使用场景 |
|----------|------|----------|
| **学生/研究人员** | 需要权威来源引用 | 写论文、做研究 |
| **产品/运营人员** | 快速获取行业信息 | 竞品分析、趋势研究 |
| **求职者** | 了解岗位技能要求 | 面试准备、职业规划 |
| **内容创作者** | 收集素材和观点 | 文章写作、视频选题 |

### 1.3 核心价值主张

> **"搜索透明，来源可溯"** - 5秒获得直接答案，而非30秒的通用回复

## 2. 用户研究

### 2.1 用户痛点访谈（摘录）

> "ChatGPT 给的信息可能是编造的，我需要知道来源"
> —— 研究生小王，写论文时

> "搜一个问题出来10条结果，我没时间一个个点开看"

> "我只是想知道美国上班时间，它给我讲了一堆劳动法历史"
> —— 求职者小张，准备面试时

### 2.2 用户旅程地图

```
遇到问题 → 搜索引擎 → 10条蓝色链接 → 逐个点击 → 整合信息 → 得到答案
                            ↓
                      耗时10-30分钟

遇到问题 → AI搜索引擎 → 直接答案 + 引用来源 → 点击验证 → 完成
                            ↓
                       耗时30秒-2分钟
```

## 3. 竞品分析

### 3.1 竞品对比

| 维度 | 传统搜索 | ChatGPT | Perplexity | 我们 |
|------|----------|---------|------------|------|
| 实时性 | ✅ | ❌ | ✅ | ✅ |
| 来源引用 | ❌ | ❌ | ✅ | ✅ |
| 意图理解 | ❌ | ✅ | ✅ | ✅ |
| 中文优化 | ✅ | ✅ | ⚠️ | ✅ |
| 自适应搜索 | ❌ | ❌ | ⚠️ | ✅ |

### 3.2 差异化策略

**vs Perplexity**：
- 更好的中文搜索策略（优先中文网站）
- 8种意图类型精准匹配
- 置信度驱动的自适应搜索

**vs ChatGPT**：
- 实时联网，信息不过时
- 每个事实都有来源引用
- 不编造，不知道就说不知道

### 5.2 核心算法

**自适应搜索逻辑**：
```
while (confidence < 0.85 && round < maxRounds) {
    // 1. 生成搜索关键词
    queries = generateQueries(query, round, intent)

    // 2. 执行搜索
    results = search(queries)

    // 3. 评估置信度
    evaluation = evaluateConfidence(query, results)

    // 4. 决定是否继续
    if (!evaluation.need_more_search) break
}
```

## 6. 成功指标

### 6.1 北极星指标

**搜索成功率** = 用户获得满意答案的搜索次数 / 总搜索次数

### 6.2 关键指标

| 指标 | 目标 | 当前 | 测量方式 |
|------|------|------|----------|
| 平均响应时间 | < 10秒 | ~8秒 | 后端日志 |
| 意图分类准确率 | > 85% | ~88% | 人工抽检 |
| 答案有用率 | > 80% | - | 用户反馈 |
| 来源覆盖率 | > 90% | ~95% | 自动检测 |

### 6.3 反馈指标

- 简单问题平均搜索轮数 ≤ 2
- 复杂问题平均搜索轮数 ≤ 4
- 用户重复搜索率 < 15%

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| API 成本过高 | 营收压力 | 1. 优化搜索轮数 2. 设置日使用上限 |
| LLM 幻觉问题 | 信任危机 | 1. 强制引用来源 2. 置信度评估 |
| 搜索 API 不稳定 | 体验下降 | 1. 多 API 备份 2. 降级策略 |
| 中文内容质量 | 用户流失 | 1. 优先中文网站 2. 来源质量评分 |
