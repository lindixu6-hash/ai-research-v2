## 一、架构演变对比

### 之前架构（串行执行）

```
用户提问
    ↓
生成关键词 [A, B, C, D, E]
    ↓
搜索 A ──→ 等待结果 ──→ 验证
    ↓
搜索 B ──→ 等待结果 ──→ 验证
    ↓
搜索 C ──→ 等待结果 ──→ 验证
    ↓
搜索 D ──→ 等待结果 ──→ 验证
    ↓
搜索 E ──→ 等待结果 ──→ 验证
    ↓
合并结果 → 分析 → 生成报告

响应时间: ~45秒
```

**问题分析**:
- 5个搜索串行执行，每个约8-10秒
- 总耗时 = 5 × 8 = 40秒+
- 用户等待体验差，可能流失

---

### 当前架构（并行执行）

```
用户提问
    ↓
生成多维度关键词 [事实+时间, 对比, site:源, 场景]
    ↓
┌─────────────────────────────────────┐
│  并行搜索所有关键词 (Promise.all)    │
│  ┌─────┐ ┌─────┐ ┌─────┐            │
│  │  A  │ │  B  │ │  C  │  同时执行   │
│  └─────┘ └─────┘ └─────┘            │
└─────────────────────────────────────┘
    ↓
URL 去重 (避免重复结果)
    ↓
合并结果 → 分析 → 生成报告

响应时间: ~10-15秒 (提升 70%)
```

**技术实现**:
```javascript
// 串行 (旧)
for (let i = 0; i < queries.length; i++) {
  await search(queries[i]);
}

// 并行 (新)
const results = await Promise.allSettled(
  queries.map(q => search(q))
);
```

## 二、今日改进清单

### 2.1 性能优化 (P0)

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 响应时间 | 45秒 | 10-15秒 | **70%** |
| 搜索策略 | 5个同义词 | 2-3个多维度 | **效率↑** |
| 结果质量 | 有重复 | URL去重 | **精准↑** |

**实现细节**:
- **并行搜索**: 使用 `Promise.allSettled` 同时执行所有搜索请求
- **URL 去重**: 用 `Set` 数据结构过滤重复链接
- **减少关键词**: 从5个同义词优化为2-3个多维度查询

---

### 2.2 搜索质量提升 (P0)

**问题诊断** (对比豆包):

| 维度 | 豆包 | 我们的搜索 | 问题 |
|------|------|-----------|------|
| 具体性 | "6月27日第一场" | "两天中的任何一天" | 不敢下判断 |
| 深度 | "河南卫视定制" | "河南省会" | 信息挖掘浅 |
| 结论 | 概率排序 | "都有可能" | 正确的废话 |

**解决方案**:

1. **搜索词策略升级**
   ```javascript
   // 旧: 同义词堆砌
   ["AI PM skills", "AI PM competencies", "AI PM requirements"]

   // 新: 多维度覆盖
   [
     "AI product manager skills 2024 2025",  // 时间限定
     "site:linkedin.com AI PM requirements", // 真实来源
     "AI PM vs traditional PM skills"        // 对比分析
   ]
   ```

2. **提示词工程**
   - 禁止"可能取决于"、"有待观察"等安全话术
   - 要求给出明确判断（即使是概率性的）
   - 融入 SOUL 原则：帮到实处、有主见、准确第一

3. **报告结构化**
   - 技能学习类：核心结论 → 具体清单 → 学习路径
   - 预测类：明确判断 → 关键事实 → 综合分析
   - 对比类：对比结果 → 差异点 → 选择建议

### 2.4 问题分类+分层回答 (PROMPT_V2) ⭐ NEW

**优化目的**：解决"小问题大报告"的用户体验问题

**问题场景**:
```
用户问："美国公司上下班时间？"
期待回答："美国公司一般是 9am-5pm，每周40小时。"
实际回答：500字复杂研究报告
```

**优化步骤**:

1. **新增问题分类Agent**
   - 5种类型：factual(事实)、prediction(预测)、howto(方法)、comparison(对比)、exploration(研究)
   - 特征识别：问"几点"→factual，问"怎么"→howto，问"vs"→comparison

2. **设计分层回答模板**
   - factual: 50-100字简洁回答
   - prediction: 概率判断 + 理由
   - howto: 步骤清单 + 资源推荐
   - comparison: 对比表格 + 选择建议
   - exploration: 深度研究报告

3. **工作流集成**
   ```
   用户提问 → 问题分类 → 生成关键词 → 搜索 → 选择模板 → 生成报告
   ```

**效果对比**:

| 问题类型 | 之前 | 之后 |
|----------|------|------|
| 事实查询 | 500字报告 | 50-100字 |
| 预测判断 | 模糊回答 | 概率+理由 |
| 方法指导 | 长篇大论 | 步骤清单 |

**预期指标**:
- 事实查询回答长度: 500字 → 50-100字
- 用户满意度: 60% → 85%

```
所有搜索均为信息检索和研究用途，不生成任何敏感内容。
```
