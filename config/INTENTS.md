# INTENTS.md - 意图分类体系

> 基于 Perplexity、Google AI Overviews 的大厂标准

---

## 核心分类（8种）

### 1. Informational-Simple（简单信息型）

**特征：** 单一事实、定义、数据查询
**关键词：** 什么是、几点、多少、哪里
**示例：**
- "美国上班时间？"
- "AI PM是什么？"
- "iPhone 16多少钱？"

**回答策略：**
- 50-100字直接回答
- 包含具体数据
- 不要复杂结构

---

### 2. Informational-Complex（复杂信息型）

**特征：** 需要详细解释、多个方面
**关键词：** 详细介绍、深入了解、全面分析
**示例：**
- "AI PM需要什么技能？"
- "区块链如何工作？"
- "TCP/IP协议详解"

**回答策略：**
- 200-500字结构化回答
- 分点列出关键信息
- 包含背景和细节

---

### 3. Navigational（导航型）

**特征：** 寻找特定网站/页面
**关键词：** 打开、官网、登录、访问
**示例：**
- "打开华为官网"
- "ChatGPT登录页面"
- "GitHub仓库地址"

**回答策略：**
- 直接提供链接
- 简要说明
- 不要过度展开

---

### 4. Transactional（交易型）

**特征：** 购买、下载、注册意图
**关键词：** 买、下载、注册、订阅、价格
**示例：**
- "买iPhone 16"
- "下载ChatGPT"
- "注册ChatGPT"

**回答策略：**
- 提供价格/渠道信息
- 不直接促销
- 中立比较

---

### 5. Commercial（商业调研型）

**特征：** 对比、推荐、选择
**关键词：** 最佳、推荐、vs、对比、哪个好
**示例：**
- "最佳CRM软件"
- "iPhone vs Android"
- "Notion vs Obsidian"

**回答策略：**
- 对比表格
- 优缺点分析
- 选择建议

---

### 6. Generative（生成型）

**特征：** 要求AI生成内容
**关键词：** 写、生成、制定、设计、创建
**示例：**
- "写求职信"
- "生成代码"
- "制定学习计划"

**回答策略：**
- 直接生成内容
- 提供多个版本
- 说明使用场景

---

### 7. Research（研究综合型）

**特征：** 深度分析、趋势研究
**关键词：** 分析、研究、综述、趋势、发展
**示例：**
- "分析2024年AI发展趋势"
- "自动驾驶技术现状"
- "量子计算应用前景"

**回答策略：**
- 完整研究报告
- 关键发现 + 数据 + 局限性
- 多维度分析

---

### 8. Multi-part（多部分型）

**特征：** 包含多个问题
**特征：** 多个问号、多个疑问词
**示例：**
- "什么是AI PM？需要什么技能？薪资多少？"
- "怎么学Python？要多久？有什么资源？"

**回答策略：**
- 分段回答
- 每个问题独立处理
- 保持结构清晰

---

## 复杂度分类

### Simple（简单）

- 单一问题
- 事实查询
- 答案在3句话内

**处理：** 1-2次搜索，快速合成

### Complex（复杂）

- 多部分问题
- 需要对比
- 需要多步推理
- 需要综合分析

**处理：** 3-5次搜索，深度合成

---

## 检测规则

```javascript
// 伪代码示例
function detectIntent(query) {
  // 1. 检查是否是多部分问题
  if (query.match(/[？?].*[？?]/)) {
    return 'multi-part';
  }

  // 2. 检查关键词
  if (query.match(/写|生成|制定|设计/)) {
    return 'generative';
  }

  if (query.match(/最佳|推荐|vs|对比/)) {
    return 'commercial';
  }

  if (query.match(/买|下载|注册|价格/)) {
    return 'transactional';
  }

  if (query.match(/打开|官网|登录/)) {
    return 'navigational';
  }

  if (query.match(/分析|研究|综述|趋势/)) {
    return 'research';
  }

  // 3. 默认为信息型
  if (query.match(/什么是|怎么|如何/)) {
    return 'informational-complex';
  }

  return 'informational-simple';
}
```
