# AI 搜索提示词优化方案 v2.0

> 基于 prompts.chat (162k stars) 和大厂提示词策略

---

## 核心问题

**当前问题：** 所有问题都用同一套复杂报告格式

**用户问：** "美国公司上下班时间？"
**期待回答：** "美国公司一般是 9am-5pm，每周40小时。部分科技公司有弹性工作制。"
**实际回答：** 复杂结构化报告（当成深度研究问题）

---

## 解决方案：问题分类 + 分层回答

### 第一步：问题分类 Agent

```javascript
QUESTION_CLASSIFIER: `你是一个问题分类专家。

任务：判断用户问题属于哪种类型，只返回类型名称。

【问题类型】
1. **factual** - 事实查询：时间、数据、定义、是什么
   - 特征：问"几点"、"多少"、"是什么"、"哪里"
   - 示例："美国上班时间"、"AI PM是什么"

2. **prediction** - 预测判断：会发生什么、哪个更好、在哪天
   - 特征：问"会不会"、"哪个"、"哪天"、"概率"
   - 示例："明天会下雨吗"、"哪个手机更好"

3. **howto** - 方法指导：怎么做、如何学、步骤
   - 特征：问"怎么"、"如何"、"第一步"
   - 示例："怎么学AI PM"、"如何减肥"

4. **comparison** - 对比分析：A vs B、区别、差异
   - 特征：问"vs"、"区别"、"差异"、"对比"
   - 示例："React vs Vue"

5. **exploration** - 深度研究：综合分析、全面了解
   - 特征：问"分析"、"研究"、"详细介绍"
   - 示例："分析一下AI行业发展趋势"

【输出格式】
只返回类型名称，不要其他内容。

输出示例：
factual
prediction
howto
comparison
exploration`
```

---

### 第二步：分层回答策略

```javascript
// 根据问题类型选择回答模板

RESPONSE_TEMPLATES: {
  // ===== 事实查询：简洁直接 =====
  factual: `# 核心答案

[1-3句话直接回答问题，包含具体数据]

## 补充信息

[如果用户需要更多背景，可以展开2-3点]

---
**风格：**
- 开门见山，第一句就给答案
- 具体数字优于模糊描述
- 不需要复杂的结构化报告
- 不要"可能取决于"等废话`,

  // ===== 预测判断：概率+理由 =====
  prediction: `# 核心判断

**判断：** [明确结论，如"大概率会"、"倾向于A"]

**理由：** [1-2句话说明核心依据]

## 分析依据

### 关键事实
[列出支撑判断的关键事实]

### 综合判断
[基于事实给出最终判断]

---
**风格：**
- 必须给出明确判断（概率性也可以）
- 不说"都有可能"
- 用事实支撑判断`,

  // ===== 方法指导：步骤清单 =====
  howto: `# 核心步骤

## 快速版（3步法）
1. [第一步]
2. [第二步]
3. [第三步]

## 详细版
### 第一步：[名称]
- 具体操作：...
- 预计时间：...
- 注意事项：...

### 第二步：[名称]
...

## 推荐资源
- 资源1：[名称 + 链接]
- 资源2：[名称 + 链接]

---
**风格：**
- 可执行的步骤清单
- 每步有具体操作
- 包含资源推荐`,

  // ===== 对比分析：对比表格 =====
  comparison: `# 核心结论

[一句话总结对比结果]

## 详细对比

| 维度 | A | B |
|------|---|---|
| 维度1 | ... | ... |
| 维度2 | ... | ... |
| 维度3 | ... | ... |

## 选择建议

- **选择A如果：** [...]
- **选择B如果：** [...]

---
**风格：**
- 表格形式清晰对比
- 给出选择建议
- 不模棱两可`,

  // ===== 深度研究：结构化报告 =====
  exploration: `# 研究报告：[主题]

## 核心摘要
[3-4句话总结核心发现]

## 关键发现
1. **发现1**：[具体描述]
2. **发现2**：[具体描述]
...

## 详细分析
### 背景信息
...
### 核心问题
...
### 趋势分析
...

## 结论
[总结性判断和建议]

---
**风格：**
- 完整的研究报告结构
- 适合复杂问题的深度分析
- 包含背景、分析、结论`
}
```

### 第三步：工作流集成

```javascript
// routes/stream.js 修改

async function executeStreamingWorkflow(query) {
  // 1. 先分类
  const questionType = await classifyQuestion(query);

  // 2. 根据类型选择策略
  const responseTemplate = RESPONSE_TEMPLATES[questionType];

  // 3. 生成搜索词（不同类型可能需要不同策略）
  const searchQueries = await generateSearchQueries(query, questionType);

  // 4. 执行搜索
  const searchResults = await batchSearch(searchQueries);

  // 5. 生成回答（使用对应模板）
  const report = await generateReport(
    query,
    searchResults,
    questionType,
    responseTemplate
  );

  return { report, questionType };
}
```

## 效果对比

### 之前（统一复杂报告）

```
用户：美国公司上下班时间？

回答：
## 研究报告：美国工作时间

## 核心摘要
在美国，工作时间因行业和地区而异...
（300字复杂报告）
```

### 之后（分类回答）

```
用户：美国公司上下班时间？

类型判断：factual（事实查询）

回答：
# 美国公司上下班时间

美国公司的标准工作时间是 **上午9点到下午5点**，每周工作约40小时（周一至周五）。

## 补充信息
- **弹性工作制**：科技公司和初创企业常采用弹性工作时间
- **午餐时间**：通常有1小时午休（不计薪）
- **加班文化**：部分行业存在加班现象，但不如东亚普遍
```
