# AI 搜索引擎迭代日志

## 2026-05-15 - 拒绝正确的废话

### 问题场景

用户用同一个问题测试了两个 AI 搜索引擎：

**问题：** 周深要在郑州开演唱会开两天，结合前几场演出，他都会唱有特色的当地歌曲，我个人喜欢《若思念便思念》这个歌，你预估会在第几天

**豆包的回答：**
- 明确判断：6月27日第一场（大概率）
- 具体理由：《若思念便思念》是河南卫视定制曲目，首场更偏重情怀主题曲
- 规律分析：总结了前几场首场vs次日的歌单规律

**我们的回答：**
- 模糊判断："有可能在两天中的任何一天"
- 泛泛理由："郑州作为河南省会"
- 没有规律分析
- 充斥"可能取决于"等安全话术

### 问题诊断

| 维度 | 豆包 | 我们的搜索 | 问题 |
|------|------|------------|------|
| 具体性 | ✅ 明确日期 | ❌ "两天中的任何一天" | 不敢下判断 |
| 深度 | ✅ 河南卫视定制 | ❌ 仅"河南省会" | 信息挖掘浅 |
| 规律 | ✅ 分析前几场 | ❌ 未分析 | 缺乏模式识别 |
| 结论 | ✅ 概率排序 | ❌ 没结论 | 正确的废话 |

### 核心问题

**我们的回答是"正确的废话"：**

1. **信息挖掘过浅** - 只停留在"河南"标签，没挖掘"河南卫视定制"这类深度关联
2. **缺乏模式识别** - 没有分析"前几场"的规律
3. **结论等于没说** - "都有可能"等于拒绝回答
4. **安全话术堆砌** - "可能取决于..."永远正确但对用户零价值
5. **过度谨慎** - 被训练成"避免说错"，但预测类问题价值恰恰在于给出判断

### 解决方案

#### 1. 搜索关键词优化

**之前：**
```javascript
// 仅覆盖事实查询
"周深郑州演唱会", "若思念便思念 周深"
```

**之后：**
```javascript
// 覆盖多维度：事实、规律、对比、背景
["周深郑州演唱会歌单", "周深演唱会首场次日规律",
 "若思念便思念 河南卫视定制", "周深巡演前几场歌单对比"]
```

#### 2. 分析提取优化

**之前：**
```javascript
// 仅提取事实
"提取有价值的信息点，包含实体名称、数字、日期"
```

**之后：**
```javascript
// 提取事实+规律
"特别关注：时间规律、数量规律、顺序规律、对比差异、因果关系"
```

#### 3. 报告生成优化

**之前：**
- 没有明确的禁止事项
- 允许"合理预测"等安全话术

**之后：**
```javascript
【绝对禁止的话术】
- "可能取决于..."
- "有待观察..."
- "都有可能..."

【必须做到】
- 给出明确判断（即使是概率性的）
- 不知道就说不知道，不要编造
```

#### 4. 融入 SOUL.md 核心原则

从优秀的 Agent 配置中借鉴：

```javascript
PRINCIPLES: `
1. 帮到实处，无需缛节
2. 要有主见，敢于下判断
3. 准确比自信重要
4. 不是客服，不是应声虫
`
```

### 新增功能

#### 搜索日志记录系统

**需求背景：**
用户希望能够记录所有搜索查询和模型回答，用于构建数据集和后续分析。

**实现方案：**

1. **日志服务** (`backend/src/services/searchLogger.js`)
   - 记录每次搜索的完整信息
   - 保存为 JSONL 格式（每行一个 JSON 对象）
   - 支持时间范围筛选

2. **日志 API** (`backend/src/routes/logs.js`)
   - `GET /api/logs/stats` - 获取统计数据
   - `GET /api/logs/export` - 导出日志
   - `GET /api/logs/download` - 下载完整文件

3. **日志管理界面** (`backend/public/logs.html`)
   - 可视化展示统计数据
   - 快捷下载：今日、本周、全部
   - 时间范围筛选

**记录内容：**
```json
{
  "timestamp": "2026-05-15T06:47:22.043Z",
  "query": "用户查询",
  "clarifyAnswers": [],
  "searchQueries": ["关键词1", "关键词2"],
  "resultsCount": 10,
  "findings": [...],
  "report": "生成的报告",
  "duration": 5000,
  "error": null
}
```

1. **前端使用相对路径** - `API_URL = '/api'`，同域部署无需 CORS

3. **nginx 代理配置** - 正确转发 API 请求

### 效果验证

部署后需要验证：

1. **搜索质量** - 用同样的问题测试，看是否给出明确判断
2. **日志记录** - 每次搜索后检查日志文件是否正确记录
3. **日志下载** - 测试今日、本周、全部下载功能
4. **时间筛选** - 选择时间范围后导出，验证筛选正确

### 经验教训

1. **提示词工程比模型更重要** - 同样的模型，不同的提示词会产生天壤之别的效果
2. **敢于下判断是价值所在** - 预测类问题的价值在于给出概率性判断，而不是永远正确但毫无价值的废话
3. **数据积累很重要** - 日志记录系统为后续优化提供数据基础
4. **同域部署简化架构** - 前后端部署在同一服务器，避免 CORS 和证书问题

### 下一步优化方向

1. **引入知识图谱** - 深度挖掘实体之间的关联关系
2. **增加搜索源** - 不只限于网页，还可以接入专业数据库、API
3. **用户反馈机制** - 允许用户对答案质量评分，用于持续优化
4. **个性化记忆** - 记住用户的偏好和搜索历史，提供更精准的结果

## 2026-05-15 晚 - 性能优化与搜索策略改进

### 问题分析

基于单条日志的深度分析发现：

| 问题 | 现象 | 影响 |
|------|------|------|
| 性能极差 | 45.7秒响应 | 用户早关页面了 |
| 搜索词冗余 | 5个词几乎同义 | 浪费配额，结果重叠 |
| 结果利用率低 | 25个结果→6个发现 | 76%的结果被浪费 |

### 对比豆包的差距

| 维度 | 豆包 | 你的项目 |
|------|------|----------|
| 响应速度 | ~3秒 | 45秒 |
| 搜索策略 | 隐式优化 | 显式冗余 |
| 报告可操作性 | 明确结论 | 泛泛而谈 |

**核心问题：** 你的项目像"研究助手"，但用户要的是"决策顾问"。

### 解决方案

#### P0: 性能优化 - 并行搜索

**之前：** 串行执行搜索，5个关键词依次执行
```javascript
for (let i = 0; i < searchResult.queries.length; i++) {
  const result = await search(queries[i]);
  // ...
}
```

**之后：** 并行执行所有搜索
```javascript
const searchResults = await batchSearch(searchResult.queries);
// 使用 Promise.allSettled 并行执行
```

**效果：** 45秒 → 预计 10-15秒

#### P0: 搜索词策略优化

**之前（同义词冗余）：**
```
❌ AI Project Management Skills
❌ Artificial Intelligence PM Competencies
❌ AI PM Technical Requirements
❌ AI PM Skill Set
❌ Project Management in AI Industry
```

**之后（多维度+限定词）：**
```javascript
✅ "AI PM skills requirements" 2024 2025  ← 加时间
✅ AI product manager vs project manager  ← 区分概念
✅ AI PM interview questions             ← 落地场景
✅ site:linkedin.com AI PM               ← 真实职位
✅ AI PM salary and career path           ← 薪资前景
```

**新增规则：**
- 每个关键词要有明确的目的，不能重复
- 优先使用具体的限定词（时间、地点、场景）
- 加入真实来源（site: 限定词）
- 不要生成同义词替换的重复关键词

#### P1: URL 去重逻辑

**问题：** PMI 同一篇文章被用了两次

**解决：** 添加 URL 级别去重
```javascript
function deduplicateResults(results) {
  const seen = new Set();
  return results.filter(result => {
    const url = result.url;
    if (seen.has(url)) return false;
    seen.add(url);
    return true;
  });
}
```

**之前：** 泛泛的报告格式

**之后：** 根据问题类型选择结构

**技能学习类：**

```
- 在线课程：[具体名称]
- 书籍：[具体名称]
```

## 2026-05-15 深夜 - 进一步性能优化

### 优化目标

基于日志分析，进一步降低响应时间：

```
34.9秒 ≈ 2s(澄清) + 2s(搜索词) + 20s(5个搜索) + 5s(分析) + 8s(报告)
```

### 优化措施

#### 1. 搜索词数量：5个 → 2-3个

**修改前：**
```javascript

```

**修改后：**
```javascript
任务：生成 2-3 个最优搜索关键词组合

```

**预期效果：** 搜索时间减少 40%

#### 2. 智能跳过澄清

**新增规则：**
```javascript
以下情况直接返回 need_clarify: false：

- 问题包含时间限定（2024、2025、最新）
- 问题已经非常具体
- 问题少于5个字且不是宽泛概念
```

**预期效果：** 减少 2秒澄清时间
